## 1. 쿠키란 무엇인가요?

쿠키는 웹사이트 서버에서 브라우저로 전송되어 기기에 저장되는 작은 텍스트 파일입니다. 쿠키를 통해 웹사이트는 사용자의 기기를 인식하고 로그인 상태 또는 언어 기본 설정과 같은 특정 정보를 기억하여 다음 방문 시 활용할 수 있습니다.

---

## 2. 쿠키 사용 방법

당사는 핵심 서비스 기능 제공에 필요한 최소한의 쿠키만 사용하며, 사용자 추적, 행동 분석 또는 광고 목적으로는 사용하지 않습니다.

### 2.1 쿠키 분류

기능 및 목적에 따라 쿠키는 다음과 같이 두 가지 범주로 분류됩니다.

| 유형 | 설명 | 동의 필요 여부 |
|------|------|-------------|
| **필수 쿠키** | 서비스의 정상적인 작동에 필수적입니다. 거부 시 일부 기능을 사용할 수 없습니다. | 아니요 |
| **선택적 쿠키** | 사용자 경험 향상을 위해 사용됩니다. 거부해도 핵심 기능에는 영향을 미치지 않습니다. | 예 |

### 2.2 쿠키 동의 메커니즘

이 웹사이트를 처음 방문하면 페이지 하단에 쿠키 동의 배너가 표시됩니다. 다음 중 하나를 선택할 수 있습니다.
- **수락**: 선택적 쿠키 설정을 허용합니다.
- **거부**: 필수 쿠키만 유지하고 기존의 선택적 쿠키는 자동으로 삭제됩니다.

선택 사항은 `cookieConsent` 쿠키에 기록되며, 365일 동안 유효합니다.

## 3. 사용하는 쿠키에 대한 세부 정보

### 3.1 필수 쿠키

다음 쿠키는 서비스의 정상적인 작동에 필수적이며, 동의 없이 설정될 수 있습니다.

| 쿠키 이름 | 목적 | 만료일 | HttpOnly | SameSite |
|------------|------|--------|----------|----------|
| `token` | 사용자 인증 JWT 토큰(로그인 상태 유지용) | 60일 | 예 | Lax |
| `link_token` | Microsoft 계정 바인딩 확인 토큰(OAuth 바인딩 프로세스에서 보안 확인용) | 10분 | 예 | Lax |
| `countdown_end` | 인증 코드 전송 버튼의 카운트다운 상태(사용자 요청 잦음 방지용) | 동적(카운트다운 기간과 동일) | 아니요 | Lax |
| `codeExpiryTime` | 인증 코드 만료 타임스탬프(프런트엔드에 인증 코드 유효 기간 표시용) | 1일 | 아니요 | Lax |
| `codeEmail` | 인증 코드와 연결된 이메일 주소(인증 코드 유효 기간 동안 컨텍스트 유지용) | 1일 | 아니요 | Lax |

### 3.2 선택적 쿠키

다음 쿠키는 사용자 경험 향상을 위해 사용되며 설정하려면 사용자의 동의가 필요합니다.

| 쿠키 이름 | 목적 | 유효 기간 | HttpOnly | SameSite |
|------------|------|--------|----------|----------|
| `selectedLanguage` | 사용자 인터페이스 언어 기본 설정으로, 다음 방문 시 선호하는 언어로 자동 전환하는 데 사용됩니다. | 365일 | 아니요 | Lax |
| `cookieConsent` | 쿠키 동의 선택 기록으로, 사용자의 기본 설정을 기억하는 데 사용됩니다. | 365일 | 아니요 | Lax |

## 4. 쿠키 보안 속성 설명

다양한 유형의 쿠키에 대해 다음과 같은 보안 속성을 구성했습니다.

### 4.1 HttpOnly 속성

| 쿠키 | HttpOnly | 설명 |
|--------|----------|------|
| `token`, `link_token` | 예 | JavaScript의 읽기를 차단하여 XSS 공격으로 토큰을 탈취할 위험을 줄입니다. |
| 기타 쿠키 | 아니요 | 프런트엔드 JavaScript가 해당 기능을 위해 읽기를 허용합니다. |

### 4.2 SameSite 속성

모든 쿠키는 `SameSite=Lax`로 설정되어 있습니다. 이는 다음과 같은 의미입니다.
- 동일 사이트 요청 시 쿠키가 전송됩니다.
- 외부 웹사이트에서 이 사이트로 이동할 때도 쿠키가 전송됩니다(GET 요청 시 허용).
- 교차 사이트 POST 요청 또는 스크립트에 의해 시작된 교차 사이트 요청은 쿠키를 전송하지 않습니다.

이 설정은 일반적인 사용자 탐색 환경에 영향을 주지 않으면서 보안을 보장합니다.

### 4.3 Secure 속성

모든 쿠키에는 `Secure` 속성이 활성화되어 있어 쿠키가 HTTPS 암호화 연결을 통해서만 전송되도록 보장하고 중간자 공격을 방지합니다.

### 4.4 Path 및 Domain 속성

- **Path**: 모든 쿠키의 Path는 `/`로 설정되어 있어 사이트 전체에서 사용할 수 있습니다.
- **Domain**: `www.nebulastudios.top`으로 설정되어 있습니다. 쿠키는 www 서브 Domain 에서만 유효하며 다른 서브 Domain 과는 분리되어 있습니다.

## 5. 제3자 쿠키

당사는 **어떠한 제3자 쿠키도 설정하지 않습니다.**

당사는 Cloudflare를 CDN 서비스 제공업체로 사용하고 있으므로, Cloudflare는 트래픽 라우팅 및 DDoS 공격 방지와 같은 기능을 위해 인프라 수준에서 기술 쿠키를 설정할 수 있습니다. 이러한 쿠키는 Cloudflare가 **독립적인 데이터 관리자**로서 설정 및 관리하며, Cloudflare의 [개인정보 보호정책](https://www.cloudflare.com/privacypolicy/)이 적용됩니다. 당사는 이러한 쿠키 설정에 대한 통제권이 없습니다.

## 6. 쿠키 관리 방법

### 6.1 당사 웹사이트를 통한 쿠키 관리

다음과 같은 방법으로 언제든지 쿠키 기본 설정을 관리할 수 있습니다.
- 브라우저 데이터를 삭제하고 웹사이트를 다시 방문하면 쿠키 동의 배너가 다시 표시됩니다.
- `cookieConsent` 쿠키를 수동으로 삭제한 후 페이지를 새로 고침합니다.

### 6.2 브라우저 설정을 통한 쿠키 관리

대부분의 브라우저는 쿠키를 보고, 삭제하고, 차단할 수 있는 기능을 제공합니다. 브라우저 설정을 통해 다음을 수행할 수 있습니다.
- 저장된 쿠키 보기
- 모든 쿠키 또는 특정 쿠키 삭제
- 모든 쿠키 또는 타사 쿠키만 차단
- 특정 웹사이트에 대한 허용 목록 설정

**주의**: 필수 쿠키를 차단하면 로그인 및 인증 코드 전송과 같은 이 서비스의 핵심 기능이 제대로 작동하지 않을 수 있습니다.

### 6.3 일반적인 브라우저 쿠키 관리 방법

| 브라우저 | 관리 경로 |
|--------|---------|
| Chrome | 설정 → 개인정보 및 보안 → 타사 쿠키 |
| Firefox | 설정 → 개인정보 및 보안 → 쿠키 및 웹사이트 데이터 |
| Safari | 환경설정 → 개인정보 → 쿠키 및 웹사이트 데이터 |
| Edge | 설정 → 쿠키 및 웹사이트 권한 → 쿠키 및 웹사이트 데이터 |
